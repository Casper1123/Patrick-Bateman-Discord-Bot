Server-wide:
- 1 Command to toggle features as a whole
    - Each feature is an optional setting
    - Running the command prevents all in one go if no optionals are passed in.
- 1 Command (global operators) to disable local fact pool for a given server ID. If left empty, it disables it for all of them.
User-wide:
- 1 command to toggle user-only features
    - Settings shared across all servers
    - Prevents things like autoreply
    - Defaults to false

User-indexed tables? UID --> BOOL / other data required for settings

Features that require supporting:
Fact -- See above!
AskPatrick -- Only usable if the chat command permissions were given to the user (ask @patrick method)
Autoreplies -- Only server-wide for sayings. User-wide AND server-wide for numbers and letters.

All slash-command permissions delegated to server owners.


How to keep track:
Seperate DB files - User & Server
Keep enabled-info stored by UID as PK; can also hash UID, might be wise for information security.
Each feature has a seperate table; means adding and removing features is as easy as killing / creating the table for it.
Couple options:
- Create entries for all users, that stores explicit on/off state
- Create entries only for users that have modified their settings
- Create entries of only the (hashed) UID of users that do not have the default settings.
